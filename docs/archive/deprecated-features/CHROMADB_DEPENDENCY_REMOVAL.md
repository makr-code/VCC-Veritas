## üìö Technical Background

### ChromaDB Architecture

ChromaDB unterst√ºtzt zwei Deployment-Modi:

1. **Local Mode (In-Process):**
   - ChromaDB l√§uft im selben Python Process
   - Ben√∂tigt `chromadb` package
   - F√ºr Development/Testing

2. **Remote Mode (HTTP API):**
   - ChromaDB Server l√§uft separat (Docker, Cloud, etc.)
   - Client nutzt nur HTTP API
   - **Ben√∂tigt NUR requests library!** ‚Üê UDS3 nutzt diesen Modus

### UDS3 Design Decision

**Warum Remote HTTP Only?**

1. **Production Setup:**
   - ChromaDB Server l√§uft als Docker Container (192.168.178.94:8000)
   - Clients verbinden √ºber HTTP API
   - Kein lokaler ChromaDB Process n√∂tig

2. **Microservices:**
   - Backend (Port 45678) ‚Üí ChromaDB Server (HTTP)
   - Ingestion Backend (Port 45679) ‚Üí ChromaDB Server (HTTP)
   - Keine lokale Instanz in den Services

3. **Skalierung:**
   - ChromaDB Server kann horizontal skaliert werden
   - Clients bleiben lightweight (nur HTTP)

**Vergleich:**

| Aspect | Local Mode | Remote Mode (UDS3) |
|--------|------------|-------------------|
| **Package Size** | 50 MB | 30 MB (-40%) |
| **Dependency** | chromadb | requests only |
| **Deployment** | Embedded | Separate Server |
| **Scaling** | Single Process | Horizontal |
| **Production** | ‚ùå Not Recommended | ‚úÖ Best Practice |
